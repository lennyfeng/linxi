# UX Usability Review — End User Perspective

> Reviewing all UI specs as daily users of the system.
> Focus: efficiency of frequent tasks, cognitive load, context switching pain.

---

## 角色 × 高频操作矩阵

| 角色 | 每天做的事 | 频率 |
|------|----------|------|
| **财务** | 录交易、对账、查余额、导入流水、看报表 | 10-30次/天 |
| **产品开发** | 看看板、推阶段、打样管理、填利润试算、提审批 | 5-15次/天 |
| **产品负责人** | 审批、看项目概况、查报价 | 3-5次/天 |
| **管理员** | 用户管理、看日志、配置同步 | 1-3次/天 |

## 🔴 严重体验问题（影响日常效率）

### 1. 财务录交易太慢 — 高频操作步骤太多

**当前流程**：打开系统 → 点左侧"Transactions" → 点"+ New Transaction" → 选类型 → Drawer打开 → 填表单 → 保存

一笔交易需要 **4 次点击才开始填写**。财务每天录 20-30 笔交易，这个摩擦是致命的。

**对比随手记**：打开 → 点"+" → 直接在页面内填写 → 保存 → 自动清空准备下一笔

**建议**：
- ✅ **全局快捷键** `Ctrl+N` → 直接弹出新交易 Drawer（不管在哪个页面）
- ✅ **连续录入模式**：保存后 Drawer 不关闭，清空表单准备下一笔（开关控制）
- ✅ **类型切换**：Drawer 内顶部放 Type 切换 Tab，不需要关闭重开
- ✅ **常用分类置顶**：Cascader 顶部显示最近 5 个使用过的分类

### 2. 无批量录入 — 逐条 Drawer 不适合批量场景

财务有时拿着一叠发票要一次性录入 10-20 笔。逐条打开 Drawer → 填写 → 保存 → 再打开，非常痛苦。

**建议**：
- ✅ 增加 **表格式批量录入**（类似 Excel 行编辑）：日期 | 类型 | 金额 | 账户 | 分类 | 摘要 → 批量提交
- 和 Drawer 单条录入并存，工具栏加 [批量录入] 按钮

### 3. 勾稽工作需要同时看两份数据 — 当前只能切 Tab

**场景**：财务拿到一张发票，要去匹配对应采购单。当前需要：
1. 打开 Invoice 列表找到发票
2. 点发票详情看金额/供应商
3. 关闭 → 切到 Purchase Orders Tab
4. 搜索同一供应商的 PO
5. 找到后点 [Link Invoice]...

来回切换 Tab 丢失上下文，效率极低。

**建议**：
- ✅ **勾稽工作台**（Reconciliation Workspace）：左侧未匹配发票列表，右侧待匹配 PO 列表，中间拖拽或选择关联。**一屏看两边**，不切 Tab
- 或者至少：Linking Modal 打开时，左侧显示发票完整信息，右侧显示候选 PO

### 4. 审批太深 — 审批人找不到入口

**场景**：产品负责人收到通知"有新审批" → 要怎么操作？
1. 点通知 → 跳转到项目详情页
2. 找到 Approvals Tab
3. 点击审批
4. 填写意见 → 提交

**建议**：
- ✅ 通知 dropdown 里直接加 **[Approve] [Reject]** 快捷按钮（简单审批不需要跳转）
- ✅ Dashboard "Pending Approvals" 卡片点进去是 **审批列表**，可以逐条处理，不是跳到各个项目
- ✅ 增加 `/approvals` 统一审批页面：所有待审批项集中展示

### 5. 看板信息不够 — 卡片内容太简略

当前卡片只有：名称、SKU、开发人、Owner、在阶段天数、目标价。

**产品开发人员最想在卡片上看到的**：
- 当前阻塞点（打样结果？还是等审批？）
- 最新利润率（通过了没？）
- 有没有新评论/动态

**建议**：
- ✅ 卡片底部增加 **状态指示器**：🔴 阻塞 / 🟡 等待 / 🟢 正常
- ✅ 鼠标 hover 卡片显示**悬浮预览**（成本、利润率、最新动态），不需要点击打开 Drawer
- ✅ 卡片角标显示未读动态数

## 🟡 中等体验问题（影响使用舒适度）

### 6. 账户余额查看不方便

财务经常需要"看一眼某账户还有多少钱"。当前需要：
导航到 Ledger → Accounts → 点击账户卡片。

**建议**：
- ✅ 交易列表页的 Stats Bar 增加 **"账户余额快览"**：可展开/折叠的迷你余额卡
- ✅ 或在 Sidebar 底部常驻显示 Top 3 账户余额

### 7. 利润试算无对比模式

产品开发想比较："卖 $25 和 $29.99 哪个更赚？"当前需要手动改价格，人脑记住上一个结果。

**建议**：
- ✅ 增加 **"价格敏感度"模式**：输入价格区间（$20~$35, step $5）→ 自动生成对比表
- ✅ 或至少支持 **并排 2 组参数对比**

### 8. 草稿态增加了认知负担

交易有 Draft / Submitted 两个状态。对于大部分场景，财务人员不需要"暂存草稿"，录完直接生效。

**建议**：
- ✅ 默认按钮改为 **[Save]**（直接提交），草稿功能放到下拉：[Save ▾] → [Save as Draft]
- 不常用的功能不要抢主按钮位置

### 9. Drawer 宽度一致性

当前设计：
- User Create Drawer: 600px
- Transaction Form Drawer: 650px
- Project Detail: 全页面
- Entity Detail: 800px

宽度不统一，用户记忆混乱。

**建议**：
- 标准化为 **两种宽度**：窄 Drawer (640px) 用于表单，宽 Drawer (800px) 用于详情
- 或者全部用 640px + 详情页改为全页面导航

### 10. 导入流程离开了交易列表

点 [Import] → 跳转到 `/ledger/import`（独立页面 4 步流程）→ 导入完 → 手动导航回交易列表。

**建议**：
- ✅ 导入完成的第 4 步增加 **自动跳转倒计时**："3 秒后返回交易列表..."
- 或者导入流程用 **全屏 Modal** 而非独立页面，关闭后自动回到之前的列表

### 11. 中文搜索最少 2 字符不合理

全局搜索设定"最少 2 字符触发"。但中文场景下 1 个字就有明确含义（"张" 搜用户、"采" 搜采购单）。

**建议**：
- ✅ 中文 1 字符即触发，英文/数字 2 字符

### 12. 通知无偏好设置

所有通知类型全部推送，随着使用增加会很嘈杂。

**建议**：
- ✅ Personal Center 增加 **通知偏好**：按类型开关（同步完成通知我不想收、审批通知必须收）

## 🟢 Nice-to-have 优化

### 13. 键盘快捷键系统
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+N` | 新建交易 |
| `Ctrl+K` | 全局搜索聚焦 |
| `Ctrl+S` | 保存当前表单 |
| `Esc` | 关闭 Drawer/Modal |
| `←` `→` | Drawer 内切换记录（上一条/下一条） |

### 14. 常用操作固定在顶部
- Sidebar 顶部增加 **"Pinned"** 区域，用户可以把常用页面拖进去
- 比如财务只需要 Transactions 和 Accounts

### 15. 操作后的即时反馈
- 创建交易后，交易列表中新行应该有 **高亮动画**（2 秒闪烁）
- 勾稽关联后，状态 badge 应该有 **过渡动画**（Not Linked → Partially Linked）

### 16. Drawer 记录切换
- 在详情 Drawer 中增加 **[← Previous] [Next →]** 按钮
- 不关闭 Drawer 直接切换到列表中的上/下一条记录
- 对审计逐条检查特别有用
